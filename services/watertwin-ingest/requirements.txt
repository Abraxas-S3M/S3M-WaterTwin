# watertwin-ingest service dependencies (pinned).

# stores them immutably (content-addressed, write-once), scans them structurally,
# and tracks them through a status lifecycle. It has NO direct database
# connection to the canonical store and NO OT network access: it talks to
# watertwin-api over the same authenticated HTTP API a human uses.

# Outbound HTTP client used to talk to watertwin-api (audit append). No OT reach.

# Identity / RBAC: JWT bearer validation against Keycloak (JWKS, RS256), reusing
# the same OIDC model as watertwin-api.

# Advisory service-event bus (packages/watertwin_events). NATS client only; the
